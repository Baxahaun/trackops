TrackOps es un sistema local de orquestacion y automatizacion operativa para proyectos con agentes IA. Prepara al agente, instala el runtime con un paso explicito, ordena el proyecto y actua como intermediario entre la idea del usuario y una ejecucion con estructura real. TrackOps is a local orchestration and operational automation system for projects built with AI agents. It prepares the agent, installs the runtime through an explicit step, organizes the project, and acts as the layer between a user's idea and a structured execution flow.

Beta publica. Si el proyecto ya esta definido, OPERA sigue por bootstrap directo. Si solo hay una idea, TrackOps deriva el arranque al agente. Public beta. If the project is already defined, OPERA continues through direct bootstrap. If there is only an idea, TrackOps routes onboarding to the agent.

## Control Operativo

Antes de iniciar cualquier tarea:
1. Lee `ops/project_control.json` como fuente de verdad.
2. Lee `ops/task_plan.md` para entender el estado.
3. Lee `ops/progress.md` para el contexto.
4. Lee `ops/findings.md` para conocer decisiones.
5. Usa `trackops status` para confirmar.

Al completar trabajo:
1. Ejecuta `trackops task complete `.
2. Ejecuta `trackops sync` para actualizar.

Metodología O.P.E.R.A.O.P.E.R.A. Methodology

                

                    OPERA es el framework operativo opcional para proyectos que necesitan un protocolo completo de ejecucion. Empieza preguntando por el perfil del usuario, el estado del proyecto y la documentacion disponible. Con eso decide si seguir por bootstrap directo o derivar discovery al agente.OPERA is the optional operating framework for projects that need a complete execution protocol. It starts by asking about the user profile, the current project state, and available documentation. From there it decides whether to continue through direct bootstrap or route discovery to the agent.

Ecosistema de SkillsSkills Ecosystem

                

                    La skill global de TrackOps prepara al agente para trabajar con el runtime y activar repositorios. El runtime se instala por separado con npm y despues cada proyecto puede anadir skills nativas para protocolos concretos como commits, changelog o arranque guiado. No compiten entre si: una prepara el sistema, otra estructura discovery, y las demas enriquecen la ejecucion diaria.The global TrackOps skill prepares the agent to work with the runtime and activate repositories. The runtime is installed separately with npm, and after that each project can add native skills for concrete protocols such as commits, changelog, or guided project start. They do not compete: one prepares the system, another structures discovery, and the rest enrich day-to-day execution.
